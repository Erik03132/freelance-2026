## 🛠 Технические решения и код

### 1. Telegram Polling & Connection
- **Проблема**: Постоянные `TelegramNetworkError: Request timeout`. Вызвано нерабочим IPv6 на сервере Timeweb.
- **Решение**: Создан подкласс `IPv4AiohttpSession`, принудительно использующий `socket.AF_INET`. 
- **Дополнение**: Обнаружена частичная блокировка IP Телеграма в сети Timeweb. Реализован `ForcedIPResolver`, перенаправляющий запросы `api.telegram.org` на гарантированно доступный IP `149.154.167.220`.
- **Результат**: Бот успешно запущен в режиме polling и моментально реагирует на команды.

### 2. Структура бота (tg_bot.py)
- **Изменения**: Перенесена инициализация `Bot` и `Dispatcher` внутрь асинхронного `main()` для предотвращения конфликтов циклов событий.
- **Логирование**: Добавлено расширенное дебаг-логирование для коллбэков `approve_draft` и `reject_draft`.

### 3. Bitrix Disk Manager
- **Проблема**: Ошибка `'NoneType' object has no attribute 'rstrip'` при попытке сохранения.
- **Решение**: Исправлен путь поиска файла `.env` в конструкторе класса. Добавлен фолбек на хардкод-URL вебхука на случай сбоя `load_dotenv`.
- **Результат**: Бот начал отвечать в чат о начале загрузки, но файлы пока физически не появляются на Диске.
