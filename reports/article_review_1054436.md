# Анализ статьи: "Как мы сократили код-ревью с двух суток до пятнадцати минут"

> Источник: https://habr.com/ru/companies/alpinadigital/articles/1054436/

## Ключевые инсайты

### 1. Три агента > Один универсал

| Агент | Роль | Инструменты |
|-------|------|-------------|
| **Библиотекарь** | Задачи, ТЗ, документация | YouTrack/Jira, документация |
| **Исследователь** | Кодовая база, зависимости | Граф связей, RAG |
| **Ревьюер** | Замечания по коду | Git, трекер |

**Вывод:** Узкая специализация даёт качество ×10 vs универсал.

### 2. Пропорция: 70% workflow, 30% агентов

```
Workflow (детерминированный):
- Библиотекарь → Исследователь → Ревьюер
- Предсказуемо, дёшево, стабильно

Автономные агенты (30%):
- Консультант для вопросов
- Поиск по документации
- Гибкость, но дороже
```

**Вывод:** Для продакшена — workflow. Автономия — только для исследовательских задач.

### 3. Четырёхэтапное внедрение

| Этап | Срок | Обязательность | Цель |
|------|------|----------------|------|
| 1. Фон | 2 недели | Нет | Снять страх |
| 2. Полноценное | 2-3 недели | Нет | Собрать фидбэк |
| 3. Перед ревью | 3-4 недели | Да | Подготовка кода |
| 4. Полная интеграция | Постоянно | Да | Первая линия |

**Вывод:** 1.5-2 месяца без бунта. Нельзя ускорять.

### 4. Что НЕ автоматизировали

| Уровень ревью | Автоматизация | Решение |
|---------------|---------------|---------|
| Механическое (60-70%) | ✅ Полная | Форматирование, типы, нейминг |
| Архитектурное | ⚠️ Советы | Человек решает |
| Написание кода | ❌ Нет | Доверие хрупкое |

**Вывод:** ИИ = советчик для высокорисковых задач. Не автор.

### 5. Метрики до автоматизации

> "Сначала замерить, сколько часов уходит. Реально ли надо автоматизировать?"

- Команда: 100 часов/месяц на ревью
- = 2.5 недели senior-разработчика
- = несколько ставок при масштабе

**Вывод:** Считай часы ДО внедрения.

## Применение к нашим проектам

### Angela (ai-eggs)

| Применение | Действие |
|------------|----------|
| **Workflow > Автономия** | Angela: 70% workflow (диалог по сценарию), 30% автономный поиск |
| **Этапы внедрения** | 4-недельный план для менеджеров |
| **Не автоматизировать** | Не писать контент автоматически — только советы |

### ai-scout

| Применение | Действие |
|------------|----------|
| **Мультиагентность** | Разделить на: Сборщика ленты + Анализатора + Дайджестера |
| **Workflow** | Детерминированный пайплайн: fetch → score → cluster → digest |
| **Метрики** | Замерить время на мониторинг до автоматизации |

### ContentCombine (agent-lab)

| Применение | Действие |
|------------|----------|
| **Три агента** | Нормализовщик + Скорер + Экспортёр |
| **70/30** | Нормализация + скоринг = workflow, кластеризация = автономия |
| **Внедрение** | 2 недели фон → 3 недели фидбэк → обязательный |

### Sherl-Research

| Применение | Действие |
|------------|----------|
| **Консультант** | Добавить агента-консультанта для вопросов по конкурентам |
| **Не автоматизировать** | Не писать отчёты автоматически — только структура + данные |

## Ключевые цитаты для использования

1. > "Код — это только выглядит как текст. На самом деле это связи."
2. > "Чем шире задача LLM, тем хуже результат."
3. > "Доверие хрупкое. Один раз советует плохое решение — потом половину проекта переписывать."
4. > "Если на старте вы строите автономного агента — через полгода будете переписывать на workflow."
5. > "Сначала замерить, потом автоматизировать."
