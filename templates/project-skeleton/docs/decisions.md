# {{PROJECT_NAME}} — Архитектурные решения (ADR)

Формат каждой записи:

```markdown
## ADR-001: Краткое название решения

- Дата: YYYY-MM-DD
- Статус: proposed | accepted | deprecated
- Контекст: почему возник вопрос
- Решение: что выбрали
- Альтернативы: что рассматривали
- Последствия: что меняется
```

## ADR-001: Шаблонное решение

- Дата: {{DATE}}
- Статус: proposed
- Контекст: Проект только что создан из шаблона.
- Решение: Использовать стандартную структуру `/projects/{{PROJECT_NAME}}`.
- Альтернативы: Собственная структура — отклонено ради единообразия.
- Последствия: Все документы, скиллы и логи размещаются по шаблону.
