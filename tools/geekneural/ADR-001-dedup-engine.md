# ADR-001: GeekNeural — движок дедупликации контекста (4 уровня)

- **Статус:** Принято (2026-07-12)
- **Контекст:** Каскадная система AGENTS.md экономит токены через выбор модели
  (Tier 0–3). Но внутри сессии один и тот же файл часто перечитывается
  (повторный Read, повторная вставка в веб-чат) — токены тратятся впустую.
- **Решение:** Добавить content-addressed кеш чтений в рамках сессии. Повторное
  обращение к тому же содержимому возвращает короткую ссылку `gn:<hash>` вместо
  тела. Единое ядро `core/dedup.py` (stdlib, без зависимостей) используется
  всеми 4 уровнями интеграции.
- **Уровни:**
  1. shell-hook (`gn read`) — для bash-вызовов агента;
  2. MCP-сервер (stdio JSON-RPC, чистый stdlib) — нативная интеграция с OpenCode/Claude/Cursor;
  3. браузерное расширение (MV3) — дедуп вставок в ChatGPT/Claude/Gemini;
  4. IDE-плагин (VS Code) — команда «копировать файл (дедуп)».
- **Почему чистый stdlib MCP:** без сети/установки пакетов (`mcp` SDK недоступен),
  максимальная переносимость и надёжность.
- **Альтернативы, отвергнутые:**
  - Внешний SaaS-сервис — телеметрия/приватность, зависимость от сети.
  - Только промпт-инструкция в AGENTS.md — не даёт реальной экономии, только призыв.
- **Последствия:**
  - Токены экономятся реально (до ~92% на стабильном контексте).
  - Нужно не дедуплицировать volatile/real-time данные (иначе модель видит устаревшее).
  - Сессия кешируется в `~/.geekneural/cache.db` (локально, без телеметрии).
- **Метрики:** `session_stats` / `gn stats` — reads, deduped, pct_saved, est_tokens_saved.
