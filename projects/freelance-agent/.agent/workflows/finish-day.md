# 🌙 Finish Day Workflow (`/finish-day`)

Этот процесс гарантирует, что ИИ сохранит полезный контекст, сделает чекпойнт и отправит данные в архив. Вызывается по команде **`/finish-day`**.

// turbo

## Шаг 1. Анализ изменений (Diff Check)
Агент анализирует:
1. Какие основные файлы были изменены: `git status`
2. Сформированы ли новые архитектурные решения (ADR).

## Шаг 2. Авто-документирование
Агент автоматически генерирует summary дня в файл `.agent/knowledge/daily_checkpoint_$(date +%Y%m%d).md`.

**Формат чекпойнта:**
- Выполненные задачи
- Открытые баги/проблемы для завтрашнего сеанса
- Изменённые зависимости
- Контекстные заметки для следующей сессии

## Шаг 3. Бэкап и пуш
1. Очистка временных логов: `rm -rf ./scratch/*`
2. Фиксация изменений в git (через commit "chore: daily checkpoint") - просим пользователя подтвердить.

## Шаг 4. Рекомендация для пользователя
В самом конце Агент выдает сообщение: `Day finished successfully! Session checkpoint created. You can close the IDE safely.`
