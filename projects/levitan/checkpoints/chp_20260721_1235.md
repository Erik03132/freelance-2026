## 2026-07-09 — Real-time migration (Яндекс Realtime)

**Статус:** DRAFT-план и ADR готовы, ждут Яндекс Cloud ключи.

### Контекст (Habr-кейс 1057176)
Сравнены три подхода для РФ:
| Подход | Задержка | Русский | Доступ | Вердикт |
|---|---|---|---|---|
| **Яндекс Realtime** | **~330 мс** | Родной, без акцента | Нативно, рублями | **Выбор для прода** |
| OpenAI Realtime | ~740 мс (под VPN) | Акцент, мужской род у женского | Заблокирован, оплата — квест | Не для РФ-прода |
| Трёхзвенка (STT→LLM→TTS) | **~4.4 с** | Зависит от STT/TTS | Нативно | Слишком медленно |

### Решение
Перевести агента на **Яндекс Realtime API**, оставив трёхзвенку (`levitan_turnbased.py`) как fallback.

### Сделано в коде
- `docs/adr/ADR-001-yandex-realtime.md` — полный ADR с архитектурой, этапами, рисками.
- `deploy/levitan_turnbased.py` — класс `RealtimeDialog` (DRAFT, GA-форма):
  - `REALTIME_MODEL` = `gpt-realtime-2.1-mini` (переключим на `yandex-rt` при ключе)
  - `session.update`: `session.type:"realtime"`, `audio.output.format`, `reasoning.effort:"low"`
  - Убран `OpenAI-Beta: realtime=v1`
  - События GA: `response.output_audio.delta`, `response.output_audio_transcript.done`, `conversation.item.input_audio_transcription.completed`
  - Поток A: capture FIFO → `input_audio_buffer.append`; Поток B: audio.delta → playback FIFO
  - Запуск: `python3 levitan_turnbased.py realtime <phone>`

### Архитектура (из статьи)
```
Клиент (телефон) → Mango callback → baresip (ext 22, greeting_bridge)
    │ SIP/RTP 8kHz PCM
    ▼
FastAPI WS-прокси (deploy/levitan_realtime.py) — держит YC_API_KEY
    │ PCM 24kHz (auresamp 8k↔24k)
    ▼
Яндекс Realtime WS (wss://llm.api.cloud.yandex.net/.../realtime)
```
**Ключевое:** backend-прокси обязателен (нет ephemeral-токенов у Яндекса).

### Блокеры до теста
1. **Яндекс Cloud API-ключ** — сервисный аккаунт с ролями `ai.speechkit-stt.user`, `ai.speechkit-tts.user`, `ai.languageModels.user`, `ai.models.user` + `YC_API_KEY`, `YC_FOLDER_ID` в `.env`. **БЛОКЕР: аккаунта нет, задача на следующую сессию**.
2. `OPENAI_API_KEY` в `.env` (для OpenAI Realtime как fallback).
3. `pip install websockets` в venv.
4. **baresip audio-мост** — playback через `aufile` из FIFO, capture через `record` в FIFO; конверсия 8k↔24k через `auresamp.so`.

### Риски
- Доступы Яндекса: с 2026-06-01 обмен OAuth на IAM закрыт — роли назначать в консоли вручную.
- baresip AEC: на ноутбучных колонках может ловить эхо — `echoCancellation` или наушники.
- Промпт: JSON-формат ломает голос — только естественная речь + function calling.
