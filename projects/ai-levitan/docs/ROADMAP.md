# ROADMAP — Levitan AI Voice Agent

> **От ручного оператора до автономного голосового агента.**
> Версия 2.0 — с фазой стабилизации инфраструктуры, обогащением CRM, юридическим блоком и реалистичной оценкой латентности.

---

## Принципы

1. **Фундамент первым.** Нельзя строить AI поверх падающего VPS и нерешённого NAT.
2. **Data-first.** Сначала данные (100 звонков) → анализ → потом автоматизация.
3. **Каждая фаза измерима.** Без метрик Фазы 1 нельзя оценить Фазы 3-5.
4. **Юридическая чистота.** ФЗ-152 и маркировка автообзвона — не опция, а условие масштабирования.
5. **Никакого перепрыгивания.** Фаза завершена и проверена → следующая.

## Фаза 1: Auto-dial + Ручной оператор + Обогащение CRM

> **Подробный план:** `docs/PHASE1_PLAN.md`

### Что это

Бот автоматически набирает номера. **Перед звонком** карточка клиента обогащается данными из базы (имя, регион, культуры). Оператор принимает звонок, ведёт диалог. После — расшифровка, LLM-анализ, сохранение в CRM с полными данными.

### Как выглядит для оператора

```
1. Telegram: «начать обзвон»
2. Бот: «📞 #1: Чиназиров Э.Р. ИП КФХ, Адыгея, Красногвардейский р-н,

3. Zoiper звонит → поднимаешь трубку
4. Гудки дозвона (3-10 сек) — Фаза 2 их уберёт
5. Клиент: «Алло?»
6. Ты: «Здравствуйте, Эдуард Рамазанович! Глобал Филдс Экспорт...»  ← обращаешься по имени
7. Диалог по скрипту (CALL_SCRIPT.txt)
8. «следующий» → через 30-90 сек карточка CRM (база + разговор)
```

## Фаза 2: Auto-dial + Мгновенное подключение (без гудков)

### Что это

Оператор **НЕ слышит гудки**. Бот набирает клиента, и только **КОГДА КЛИЕНТ ОТВЕТИЛ** — Zoiper звонит оператору. Оператор поднимает — сразу говорит с клиентом.

> **Зависит от Фазы 0** (решённый NAT). Без неё baresip не примет звонок.

### Как выглядит для оператора

```
1. Telegram: «начать обзвон»
2. Бот: «📞 #1: Чиназиров Э.Р. ...»
3. (в фоне: baresip набирает клиента → клиент ответил → перевод на оператора)
4. Zoiper звонит → поднимаешь → сразу клиент: «Алло?»
5. Ты: «Здравствуйте, Эдуард Рамазанович!...»
```

### Технические компоненты

| Компонент | Детали | Статус |
|-----------|--------|--------|
| baresip (ext 23) | SIP-клиент, ctrl_tcp порт 4444 | ✅ Регистрируется |
| NAT решён (Фаза 0) | baresip на VPS с публичным IP | ⏳ Зависит от Фазы 0 |
| SIP-перевод | baresip → ext 22 (Zoiper) | ❌ Не настроен |
| controller.py | ответ → мгновенный transfer, без приветствия | ✅ Переписан |
| Интеграция в бот | callback ext 23 вместо ext 22 | ❌ Не сделано |

## Фаза 3: Smart Assist — AI-подсказки оператору (пост-фактум + live)

### Что это

AI **слушает разговор и показывает подсказки** оператору. Начинаем с near-real-time (расшифровка фрагментами), а не с чистого streaming — так реалистичнее.

> ⚠️ **Технический риск:** Real-time русский STT на телефонном аудио 8кГц — сложная задача.
> Vosk даёт слабое качество, faster-whisper "streaming" — это чанки, а не настоящий поток.
> Начинаем с чанков по 3-5 сек, не обещаем мгновенность.

### Как выглядит для оператора

```
1. Ты говоришь с клиентом
2. На экране (Telegram / HTML overlay):
   ┌────────────────────────────────────┐
   │ 🟢 КЛИЕНТ: «А цена какая?»          │
   │ 💡 СКАЖИ: «Зависит от культуры и    │
   │    объёма. Назовите параметры.»     │
   │ 📋 ИЗ БАЗЫ: Чиназиров, зерновые     │
   └────────────────────────────────────┘
3. Ты говоришь подсказку (или своё)
```

### Технические компоненты

| Компонент | Технология | Риск |
|-----------|-----------|------|
| Near-real-time STT | faster-whisper, чанки 3-5 сек + VAD | Средний (8кГц телефон) |
| VAD | Silero VAD / webrtcvad | Низкий |
| Контекст диалога | Буфер реплик + данные из базы | Низкий |
| LLM-ассистент | DeepSeek Chat V3 (OpenRouter) | Низкий |
| Интерфейс | HTML overlay на localhost (лучше Tkinter) | Низкий |

### Задачи фазы 3

- [ ] **Near-real-time STT**: чанки 3-5 сек, VAD режет по тишине
  - Замер реальной задержки на телефонном аудио — честно, без обещаний
- [ ] **VAD**: конец речи клиента по тишине >1.5 сек
- [ ] **LLM-ассистент**:
  - Промпт: скрипт GFE + база знаний (цены, условия, возражения) + данные из базы контактов
  - Вход: реплика клиента + контекст + карточка из базы
  - Выход: 1-2 коротких предложения
- [ ] **Интерфейс подсказок**: HTML overlay на localhost (обновление через WebSocket/SSE)
- [ ] **Интеграция с dialer_bot.py**: фоновый процесс ассистента
- [ ] Тест на 10 звонках → замер реальной задержки

### Критерии успеха
- Подсказка появляется <5 сек после реплики клиента (реалистично для 8кГц)
- Подсказки релевантны (>70% полезных по оценке оператора)
- Оператор быстрее отвечает на типовые возражения

## Фаза 4: Turn-based AI-диалог (⚠️ требует realtime API)

### Что это

AI ведёт **начальную квалификацию**: задаёт вопросы, слушает, фиксирует. Оператор слышит и может вмешаться. После квалификации AI завершает (нет интереса) или передаёт оператору (есть объём).

> ⚠️ **Ключевой пересмотр:** Свой каскад STT→LLM→TTS на русском даёт паузу **3-6 сек** между репликами.
> Клиент решит, что связь оборвалась. **Для естественного диалога нужен realtime API уже здесь**,
> а не только в Фазе 5. Свой каскад пригоден максимум для 1-2 простых вопросов.
>
> ⚠️ **Юридически:** как только AI говорит сам — по закону нужно раскрыть, что это робот
> («Вас приветствует голосовой помощник компании...»). См. раздел «Юридический блок».

### Как выглядит

```
1. Бот набирает → клиент отвечает
2. AI (с раскрытием): «Здравствуйте, это голосовой помощник Глобал Филдс.
   Подскажите, у вас есть зерновые к продаже?»
   Клиент: «Да, 500 тонн пшеницы»
   AI: «Какого класса? Где находится?»
   Клиент: «4 класс, Краснодарский край»
   AI: «Соединю со специалистом, одну секунду.»
3. Zoiper звонит → оператор → к готовому клиенту
```

ИЛИ (не целевой):
```
   Клиент: «Нет, только посеяли»
   AI: «Понял. Когда планируете реализацию?»
   Клиент: «В сентябре»
   AI: «Поставлю напоминание, менеджер свяжется ближе к дате.»
   (звонок завершён, оператор не привлекался)
```

### Технические компоненты

| Компонент | Технология | Риск |
|-----------|-----------|------|
| Диалоговый движок | **realtime API рекомендован** (не свой каскад) | Высокий, если каскад |
| Бридж | baresip держит линию весь звонок | Средний |
| VAD + барж-ин | клиент перебивает → AI замолкает | Высокий |
| TTS | кешированные фразы (приветствие, «соединю») | Низкий |
| Перехват оператором | кнопка «я скажу» → смена аудио-источника | Средний |
| Эскалация | Mango callback ext 22, baresip завершает | Средний |

### Задачи фазы 4

- [ ] **Решение по движку**: realtime API (Gemini Live/OpenAI) vs ограниченный каскад
- [ ] **Юридическое раскрытие**: обязательная фраза «это голосовой помощник»
- [ ] **Диалоговый промпт**: роль менеджера GFE, максимум 5 реплик → эскалация/завершение
- [ ] **Turn-based цикл** с обработкой перебивания (barge-in)
- [ ] **TTS-кеш**: частые фразы предгенерированы
- [ ] **Перехват оператором**: baresip меняет источник TTS → микрофон
- [ ] **Интеграция с CRM**: AI сохраняет данные квалификации
- [ ] Тест на 20 звонках

### Критерии успеха
- AI раскрывает, что он робот (юридическое требование)
- AI корректно квалифицирует в 60%+ случаев
- Пауза между репликами <1.5 сек (только с realtime API)
- Оператор подключается только к целевым лидам
- Barge-in работает: клиент перебивает → AI замолкает <0.5 сек

## Фаза 5: Real-time AI Voice Agent

### Что это

Полностью автономный агент. Оператор **мониторит дашборд** с несколькими звонками. AI ведёт диалог (задержка <1 сек), обрабатывает возражения, эскалирует сложное.

### Как выглядит

```
┌─────────────────────────────────────────────────┐
│ Levitan Dashboard                    13:25      │
├─────────────────────────────────────────────────┤
│ 🟢 +79001234567  ООО АгроМир     03:12         │
│    AI: «...какой объём?» / Клиент: «500 тонн»   │
│    Статус: квалификация                          │
├─────────────────────────────────────────────────┤
│ 🔴 +79001112233  КФХ Петров       00:32         │
│    ⚠️ ЭСКАЛАЦИЯ — возьми трубку!  [ПРИНЯТЬ]     │
└─────────────────────────────────────────────────┘
```

### Архитектура (варианты)

**Вариант A: Свой каскад (low latency)**
```
Mango SIP → baresip → RTP → Whisper live → LLM streaming → edge-tts → RTP
```
- Плюс: полный контроль, ~$0.01/звонок
- Минус: latency 1-3 сек, на русском тяжело достичь естественности

**Вариант B: Внешний Voice API**
```
Mango SIP → baresip/бридж → WebSocket → Voice API
```
- OpenAI Realtime: ~$0.30/мин, latency <500 мс
- Gemini Live: ~$0.10/мин, дешевле, **но нужен VPN из РФ** (риск блокировки)

**Вариант C: Гибрид**
- Voice API для быстрой реакции + каскад для function calling

**Вариант D: xAI Voice Agent Builder (no-code)**
> Документ: `docs/XAI_VOICE_AGENT_ANALYSIS.md`
- Плюс: no-code, sub-second, встроенные инструменты, $25 кредит
- Минус: ~$0.25/звонок, бета, зависимость от xAI

### Задачи фазы 5

- [ ] **Выбор Voice API** (с учётом доступности из РФ)
- [ ] **SIP-WebSocket бридж**: baresip ↔ Voice API
- [ ] **Function calling**:
  - `save_to_crm(...)`, `escalate_to_operator(...)`, `schedule_callback(...)`
- [ ] **Дашборд мониторинга** (HTML/JS на localhost, live-транскрипт)
- [ ] **Параллельные звонки**: 3-5 одновременных
- [ ] **A/B тест**: Фаза 1 (ручной) vs Фаза 5 (AI) на 50 звонках
- [ ] Замер конверсии и стоимости против бейзлайна

### Критерии успеха
- AI завершает >70% звонков без оператора
- Конверсия не ниже бейзлайна Фазы 1
- Стоимость AI-звонка < минуты оператора
- Задержка <1 сек (субъективно естественно)
- Один оператор мониторит 3-5 звонков

## Порядок отработки

Каждая фаза завершена и проверена → следующая. Юридический блок и метрики — сквозные.

```
Фаза 0 (инфраструктура)
  → Фаза 1 (100 звонков + бейзлайн + обогащение CRM)
    → Фаза 2 (10 тестов, NAT решён)
      → Фаза 3 (10 тестов, near-real-time STT)
        → Фаза 4 (20 тестов, realtime API + раскрытие)
          → Фаза 5 (A/B на 50 звонках vs бейзлайн)
```

### Критические зависимости
- Фаза 2 **блокирована** Фазой 0 (NAT)
- Фаза 4 **требует** realtime API (свой каскад не даст естественности на русском)
- Фазы 4-5 **требуют** юридического раскрытия робота
- Все сравнения качества **требуют** бейзлайна Фазы 1
