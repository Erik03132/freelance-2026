## Фаза 1: Auto-dial + Ручной оператор

### Что это

Бот автоматически набирает номера из базы. Оператор принимает звонок в Zoiper, слышит гудки дозвона, говорит приветствие, ведёт диалог. После звонка — автоматическая расшифровка, LLM-анализ, сохранение в CRM.

### Как выглядит для оператора

```
1. Telegram: «начать обзвон»
2. Бот: «📞 Звоню #1: ООО АгроМир, Ростовская обл., +7900...»
3. Zoiper звонит → поднимаешь трубку
4. Слышишь гудки дозвона до клиента (3-10 сек)
5. Клиент: «Алло?»
6. Ты: «Здравствуйте. Глобал Филдс Экспорт. Закупаем зерновые...»
7. Диалог по скрипту (CALL_SCRIPT.txt)
8. Кладёшь трубку → Telegram: «следующий»
9. Через 30-90 сек: карточка CRM от LLM
10. Повтор с пункта 2
```

### Технические компоненты

| Компонент | Файл | Статус |
|-----------|------|--------|
| Telegram бот | `scripts/dialer_bot.py` | ✅ Готов |
| База контактов | `data/campaigns/csv/all_contacts_2026.csv` | ✅ Готов |
| Mango callback ext 22 | API `commands/callback` | ✅ Работает |
| Zoiper (ext 22) | v1000@vpbx400374818.mangosip.ru | ✅ Работает |
| Запись разговора | Mango recording API | ✅ Работает |
| STT | faster-whisper base (VPS или локально) | ✅ Готов |
| LLM-извлечение | DeepSeek Chat V3, OpenRouter | ✅ Готов |
| CRM | CSV + JSON, `data/call_results/` | ✅ Готов |

## Фаза 2: Auto-dial + Робот-приветствие

### Что это

Оператор **не слышит гудки** и **не говорит приветствие**. Бот набирает клиента, робот (baresip) произносит приветствие, и только **после этого** Zoiper звонит оператору. Оператор поднимает трубку — и сразу говорит с уже разогретым клиентом.

### Как выглядит для оператора

```
1. Telegram: «начать обзвон»
2. Бот: «📞 Звоню #1...»
3. (в фоне: baresip автоответил → Mango звонит клиенту → клиент ответил → greeting.wav играет)
4. Zoiper звонит → поднимаешь трубку
5. Сразу клиент (уже услышал приветствие): «Да, слушаю»
6. Ты: «Подскажите, у вас есть объёмы к продаже?»
7. Диалог → «следующий» → повтор
```

### Технические компоненты

| Компонент | Детали | Статус |
|-----------|--------|--------|
| baresip (ext 23) | SIP-клиент, регистрация по TCP | ✅ Регистрируется |
| greeting.wav | TTS (edge-tts, 8kHz mono) | ✅ Готов |
| Аудио-пайплайн | aufile → PCMA → RTP | ✅ Проигрывается |
| Mango callback ext 23 | `from.extension=23` | ❌ ORIGINATOR_CANCEL |
| SIP-перевод | baresip → ext 22 (Zoiper) | ❌ Не настроен |
| Интеграция в бот | callback ext 23 вместо ext 22 | ❌ Не сделано |

### Задачи фазы 2

- [ ] **Починить callback из ext 23**: разобраться почему Mango отменяет звонок
  - Гипотеза 1: baresip отвечает слишком быстро (answermode=early)
  - Гипотеза 2: RTP не доходит (NAT, SDP с приватным IP)
  - Гипотеза 3: нужен STUN для media NAT traversal
- [ ] **SIP-перевод**: baresip → Zoiper
  - Вариант A: SIP REFER (baresip команда `/call/transfer sip:22@vpbx400374818.mangosip.ru`)
  - Вариант B: два последовательных callback (ext 23 для приветствия, затем ext 22 для оператора)
- [ ] **Переключить dialer_bot.py** на callback из ext 23
- [ ] Протестировать полный цикл с тестовым телефоном
- [ ] 10 тестовых звонков → замер конверсии

### Критерии успеха

- Оператор НЕ слышит гудки дозвона
- Клиент слышит приветствие (greeting.wav) до подключения оператора
- Оператор подключается ТОЛЬКО к живым клиентам
- Задержка между ответом клиента и звонком Zoiper <3 сек

## Фаза 3: Smart Assist — AI-подсказки оператору

### Что это

Пока оператор говорит с клиентом, AI в реальном времени **слушает разговор, анализирует речь клиента и показывает подсказки** на экране. Оператор видит: что ответить на возражение, какую информацию уточнить, какую цену назвать.

### Как выглядит для оператора

```
1. Бот набирает → робот-приветствие → Zoiper звонит
2. Ты говоришь с клиентом
3. На экране (Telegram / floating окно):
   ┌────────────────────────────────────┐
   │ 🟢 КЛИЕНТ: «А цена какая?»          │
   │ 💡 СКАЖИ: «Зависит от культуры и    │
   │    объёма. Назовите параметры —     │
   │    посчитаем.»                      │
   │                                     │
   │ 🟢 КЛИЕНТ: «У меня 500 тонн пшеницы»│
   │ 💡 УТОЧНИ: «Какой класс? Где        │
   │    находится? FCA или CPT?»         │
   └────────────────────────────────────┘
4. Ты говоришь подсказку (или своё)
5. Диалог продолжается
```

### Технические компоненты

| Компонент | Технология |
|-----------|-----------|
| Real-time STT | faster-whisper streaming / Vosk / Whisper API |
| VAD | Voice Activity Detection (silence detection) |
| Контекст диалога | Буфер последних N реплик |
| LLM-ассистент | DeepSeek Chat V3 (OpenRouter) |
| Интерфейс | Telegram сообщения / Tkinter floating окно / HTML overlay |
| Системный промпт | Инструкция ассистента: скрипт, возражения, цены |

### Задачи фазы 3

- [ ] **Потоковый STT**: расшифровка речи клиента в реальном времени
  - faster-whisper с VAD: ждать тишину → расшифровать фрагмент
  - Или: Google Speech-to-Text streaming API
- [ ] **VAD**: определение конца речи клиента по тишине >1.5 сек
- [ ] **LLM-ассистент**:
  - Промпт: скрипт GFE + база знаний (цены, условия, возражения)
  - На вход: последняя реплика клиента + контекст диалога
  - На выход: 1-2 коротких предложения для оператора
  - Задержка: <2 сек от конца речи клиента до подсказки
- [ ] **Интерфейс подсказок**:
  - Вариант A: Telegram — бот шлёт сообщение с подсказкой
  - Вариант B: отдельное окно (Tkinter/PyQt) всегда поверх Zoiper
  - Вариант C: HTML-страница на localhost, открытая в браузере
- [ ] **Интеграция с dialer_bot.py**: фоновый процесс ассистента
- [ ] Тест на 10 звонках

### Критерии успеха

- Подсказка появляется <2 сек после конца речи клиента
- Подсказки релевантны (не менее 70% полезных)
- Оператор тратит меньше времени на обдумывание ответа
- Средняя длительность звонка не увеличилась

## Фаза 4: Turn-based AI-диалог

### Что это

AI сам ведёт **начальную квалификацию** клиента: задаёт вопросы, слушает ответы, фиксирует данные. Оператор **слышит разговор** и может **вмешаться в любой момент**. После квалификации AI либо завершает звонок (нет интереса), либо передаёт оператору (есть объём).

### Как выглядит для оператора

```
1. Бот набирает → робот-приветствие → клиент отвечает
2. Ты слышишь в наушниках:
   AI: «Подскажите, у вас есть объёмы зерновых к продаже?»
   Клиент: «Да, 500 тонн пшеницы»
   AI: «Какого класса? Где находится партия?»
   Клиент: «4 класс, Краснодарский край»
   AI: «Сейчас соединю со специалистом. Одну секунду.»
3. Zoiper звонит → поднимаешь → сразу к готовому клиенту
4. Продолжаешь: «Добрый день! Слышал, у вас 500 тонн 4 класса...»
```

ИЛИ если клиент не в целевой:

```
   AI: «Подскажите, у вас есть объёмы к продаже?»
   Клиент: «Нет, мы только посеяли»
   AI: «Понял. Когда планируете реализацию?»
   Клиент: «В сентябре»
   AI: «Хорошо, я поставлю напоминание. Менеджер свяжется с вами ближе к дате.»
   (звонок завершён, оператор не привлекался)
```

### Технические компоненты

| Компонент | Технология |
|-----------|-----------|
| Бридж | baresip держит линию на протяжении всего звонка |
| VAD | Определение конца речи клиента |
| STT | Потоковая расшифровка |
| LLM-диалог | Системный промпт: роль менеджера GFE, скрипт квалификации |
| TTS | edge-tts (быстрый, локальный / кешированные фразы) |
| Звуковой поток | TTS → WAV → baresip аудио-источник → RTP в линию |
| Перехват оператором | Кнопка/команда «я скажу» → baresip меняет аудио-источник на микрофон |
| Эскалация | AI вызывает Mango callback ext 22, baresip завершает звонок |

### Задачи фазы 4

- [ ] **Диалоговый LLM-промпт**:
  - Роль: менеджер GFE
  - Цель: выяснить культуру, объём, регион, класс, цену
  - Правила: не спорить, не навязываться, фиксировать возражения
  - Максимум 5 реплик AI, затем эскалация или завершение
- [ ] **Turn-based цикл**:
  - AI говорит → ждёт тишину 2 сек → слушает клиента → STT → LLM → TTS → говорит
  - Обработка перебивания: если клиент говорит во время речи AI → AI замолкает
- [ ] **TTS-кеширование**: частые фразы предгенерированы (приветствие, «соединю со специалистом», «спасибо»)
- [ ] **Механизм перехвата**: оператор нажимает кнопку → baresip переключает аудио-источник с TTS-файла на микрофон
- [ ] **Интеграция с CRM**: AI сам сохраняет данные квалификации
- [ ] Тест на 20 звонках

### Критерии успеха

- AI корректно квалифицирует клиента в 60%+ случаев
- Оператор подключается только к целевым лидам
- Среднее время звонка с AI: <90 сек
- Оператор слышит диалог AI и может вмешаться за <2 сек

## Фаза 5: Real-time AI Voice Agent

### Что это

Полностью автономный голосовой агент. Оператор не участвует в разговоре — только **мониторит дашборд** с несколькими одновременными звонками. AI ведёт диалог в реальном времени (задержка <500 мс), обрабатывает возражения, фиксирует данные. При сложных ситуациях AI эскалирует на оператора.

### Как выглядит для оператора

```
Дашборд (браузер / Telegram):

┌─────────────────────────────────────────────────┐
│ Levitan Dashboard                    13:25      │
├─────────────────────────────────────────────────┤
│ 🟢 +79001234567  ООО АгроМир     03:12         │
│    AI: «...какой объём?»                        │
│    Клиент: «500 тонн»                            │
│    Статус: квалификация                          │
├─────────────────────────────────────────────────┤
│ 🟢 +79007654321  ИП Иванов        01:45         │
│    AI: «...где находится?»                       │
│    Клиент: «Ростовская обл.»                     │
│    Статус: квалификация                          │
├─────────────────────────────────────────────────┤
│ 🔴 +79001112233  КФХ Петров       00:32         │
│    AI: «Сейчас соединю со специалистом»          │
│    ⚠️ ЭСКАЛАЦИЯ — возьми трубку!                │
│    [ПРИНЯТЬ ЗВОНОК]                              │
└─────────────────────────────────────────────────┘
```

### Архитектура (варианты)

**Вариант A: Свой каскад (low latency)**
```
Mango SIP → baresip → RTP → Whisper live → LLM streaming → edge-tts → RTP → baresip
```
- Плюс: полный контроль, нет внешних зависимостей
- Минус: latency 1-3 сек, сложно достичь <500 мс

**Вариант B: Внешний Voice API**
```
Mango SIP → baresip/бридж → WebSocket → Voice API (OpenAI Realtime / Gemini Live)
```
- Плюс: latency <500 мс из коробки
- Минус: стоимость ($0.06-0.12/мин), зависимость от API
- OpenAI Realtime: $0.06/мин вход + $0.24/мин выход ≈ $0.30/мин
- Gemini Live: дешевле, но нужен VPN из РФ

**Вариант C: Гибрид**
- Voice API для быстрой реакции (приветствие, базовые ответы)
- Каскад STT→LLM→TTS для сложных ответов и function calling

**Вариант D: xAI Voice Agent Builder (no-code) 🔵 NEW**
> Документ: `docs/XAI_VOICE_AGENT_ANALYSIS.md`
> Статус: Бета-версия доступна, сохранено для использования в Фазе 5

```
Mango SIP → baresip/бридж → WebSocket → xAI Voice Agent API
```
- Плюс: no-code конструктор, <2 мин на создание агента
- Плюс: sub-second латентность, встроенные инструменты (web search, function calling)
- Плюс: бесплатный номер телефона
- Плюс: SOC 2 Type II, HIPAA, GDPR compliant
- Минус: стоимость $0.05/мин ($0.25/звонок)
- Минус: зависимость от xAI API
- Минус: бета-версия (может быть нестабильна)
- Бесплатный кредит: $25 при регистрации
- **Рекомендация:** Использовать для пилота и A/B тестирования в Фазе 5

### Задачи фазы 5

- [ ] **Выбор Voice API**: OpenAI Realtime vs Gemini Live vs xAI Voice Agent Builder vs свой каскад
- [ ] **SIP-WebSocket бридж**: baresip ↔ Voice API
  - Преобразование SIP/RTP в WebSocket аудио-поток
  - Обработка переподключений, silence detection
- [ ] **Function calling**:
  - `save_to_crm(product, volume, region, price)` — сохранить лид
  - `escalate_to_operator(reason)` — перевести на оператора

- [ ] **Дашборд мониторинга**:
  - Web-интерфейс (HTML/JS на localhost)
  - Список активных звонков, статус, транскрипт в реальном времени
  - Кнопка «принять звонок» для эскалации
- [ ] **Параллельные звонки**: масштабирование до 3-5 одновременных

- [ ] Замер конверсии и стоимости

### Критерии успеха

- AI завершает >70% звонков без участия оператора
- Конверсия в лид не ниже, чем у ручного оператора
- Стоимость AI-звонка < стоимости минуты оператора
- Задержка ответа AI <1 сек (субъективно «естественно»)
- Один оператор мониторит 3-5 одновременных звонков
