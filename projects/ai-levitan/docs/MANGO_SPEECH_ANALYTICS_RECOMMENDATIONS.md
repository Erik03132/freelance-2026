## 🎯 Применение для ai-levitan

### 1. ИИ Помощник: Извлечение лидов (замена текущего LLM)

**Текущая реализация:** `smart_dialer.py` → DeepSeek через OpenRouter → JSON

**Рекомендация:** Использовать ИИ Помощника Mango

**Промпт для ИИ Помощника:**
```
Проанализируй диалог менеджера (Иван, «Глобал Филдс Экспорт») с клиентом (сельхозпроизводитель).

Определи:
1. Статус: лид/отказ/перезвон/нет интереса/wrong_number
2. Уровень интереса: high/medium/low/none
3. Культура: список упомянутых культур
4. Объём: если назвал
5. Регион: если упомянул
6. Возражения: список
7. Следующее действие: 1 предложение

Ответ: структурированный JSON.
```

**Преимущества:**
- Не нужен OpenRouter ($0)
- Встроенная STT (не нужен faster-whisper)
- Автоматический анализ сразу после расшифровки
- Интеграция с отчётами Mango

**Недостатки:**
- Задержка расшифровки (часы, не секунды)
- Лимит 100 минут/сутки в демо
- Нет real-time анализа

---

### 2. ИИ Конспекты: Автоматические саммари

**Настройка:**
1. Включить ИИ Конспекты в Настройках
2. Выбрать «Пользовательский промпт»
3. Вставить промпт:

```
Краткое резюме звонка (3-5 предложений):
- Цель звонка
- Что обсудили (культура, объём, регион)
- Итог: лид/отказ/перезвон
- Следующий шаг
```

**Результат:** Каждый звонок получает автоматическое резюме в плеере Mango.

---

### 3. ИИ Тематики: Автоклассификация

**Рекомендуемые тематики:**

| Тематика | Промпт | Использование |
|----------|--------|---------------|
| **Есть объём** | Клиент упомянул конкретный объём зерна (тонны, вагоны) | Фильтр лидов |
| **Готов продать** | Клиент выразил готовность продать зерно в ближайшее время | Горячие лиды |
| **Нет объёма** | Клиент не имеет объёмов или только посеял | Отсев |
| **Ценовой вопрос** | Клиент спрашивает или обсуждает цену | Интерес к сделке |
| **Нужна доставка** | Клиент обсуждает логистику или доставку | Потребность в сервисе |
| **Конкуренты** | Клиент упоминает других покупателей | Аналитика рынка |
| **Перезвон позже** | Клиент просит перезвонить через определённый срок | Планирование |
| **Не интересно** | Клиент отказался или не заинтересован | Отсев |

---

### 4. Тематики (ключевые слова): Быстрый старт

**Готовые тематики Mango (рекомендуется включить):**

1. ✅ Клиент отказался от покупки
2. ✅ Сотрудник уточнил стоимость
3. ✅ Сотрудник закрывает сделку
4. ✅ Проблемная ситуация у клиента
5. ✅ Сравнение с конкурентами

**Пользовательские тематики:**

| Тематика | Ключевые слова (термы) |
|----------|----------------------|
| **Зерновые** | пшеница, ячмень, кукуруза, зерно, зерновые |
| **Масличные** | подсолнечник, рапс, соя, масличные |
| **Бобовые** | горох, нут, чечевица, бобовые |
| **Объём** | тонн, тонны, вагонов, партия, объём |
| **Цена** | цена, стоимость, сколько стоит, дорого, дёшево |
| **Доставка** | доставка, привезти, отгрузка, логистика |
| **Регион** | область, край, район, регион, находится |

### 5. Чек-лист: Контроль скрипта

**Критерии для оценки звонка:**

| Критерий | Тип | Баллы |
|----------|-----|-------|
| Поздоровался | ✅ Положительный | +2 |
| Представился (имя, компания) | ✅ Положительный | +2 |
| Задал вопрос о потребностях | ✅ Положительный | +3 |
| Уточнил культуру | ✅ Положительный | +2 |
| Уточнил объём | ✅ Положительный | +2 |
| Уточнил регион | ✅ Положительный | +1 |
| Предложил условия | ✅ Положительный | +3 |
| Договорился о следующем шаге | ✅ Положительный | +3 |
| Не поздоровался | ❌ Отрицательный | -3 |
| Не представился | ❌ Отрицательный | -2 |
| Навязывался | ❌ Отрицательный | -2 |
| Грубил | ❌ Отрицательный | -5 |

### 6. ИИ Помощник: Анализ возражений

**Промпт:**
```
Проанализируй диалог. Определи:
1. Какие возражениязвучали от клиента (список)
2. Как менеджер их обработал
3. Оценка обработки: хорошо/нормально/плохо
4. Рекомендация: что можно было бы ответить лучше

Максимум 5 возражений, кратко.
```

## 🔧 Интеграция с текущей системой

### Вариант А: Полная замена (рекомендуется для начала)

```
Текущий пайплайн:
  Mango callback → Zoiper → запись → VPS → faster-whisper → DeepSeek → CRM

Новый пайплайн:
  Mango callback → Zoiper → запись → Mango Речевая аналитика → CRM
```

**Шаги:**
1. Подключить Речевую аналитику (демо: 7 дней, 100 мин/сутки)
2. Включить запись разговоров (многоканальная)
3. Настроить ИИ Помощника для извлечения лидов
4. Настроить ИИ Тематики для автоклассификации
5. Экспорт данных через отчёты или API

### Вариант Б: Гибрид (рекомендуется для production)

```
Текущий пайплайн (real-time):
  Mango callback → Zoiper → запись → VPS → faster-whisper → DeepSeek → CRM

Mango аналитика (retrospective):
  Mango Речевая аналитика → ИИ Помощники → Отчёты → Дашборд
```

**Преимущества:**
- Real-time анализ сохраняется (текущий пайплайн)
- Mango добавляет глубокую аналитику (чек-листы, тематики, инсайты)
- Двойная проверка качества

## 📝 Заключение

Mango Речевая аналитика может **значительно упростить** текущий пайплайн ai-levitan:

1. **Убрать** зависимость от VPS (faster-whisper + DeepSeek)
2. **Автоматизировать** классификацию лидов (ИИ Тематики)
3. **Добавить** контроль качества (Чек-листы)
4. **Получить** глубокую аналитику (Ловец инсайтов)

**Рекомендация:** Начать с демо-режима параллельно с текущей системой. Сравнить результаты и принять решение о миграции.
