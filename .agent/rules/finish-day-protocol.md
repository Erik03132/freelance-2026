# 🛰️ Протокол FINISH-DAY

## Триггер
Когда пользователь пишет **"finish-day"**, **"финиш"**, **"заканчиваем"**, **"конец дня"** или что-то похожее — выполни этот протокол ПОЛНОСТЬЮ.

## Алгоритм (строго по порядку)

### Шаг 1: Создай чекпоинт и стенограмму
1. Перезапиши основной файл `~/freelance-2026/chp.md` (для быстрой проверки).
2. Создай уникальный файл чекпоинта `~/freelance-2026/checkpoints/chp_YYYYMMDD_HHMM.md`.
3. **СОЗДАЙ СТЕНОГРАММУ СЕССИИ**: Создай файл `~/freelance-2026/reports/report-day_YYYY-MM-DD_HHMM.md`.

#### Структура Стенограммы (report.md):
```markdown
# 📝 СТЕНОГРАММА СЕССИИ: [ДАТА] [ВРЕМЯ]
## 🎯 Цели сессии
- [Кратко, что планировали сделать]

## 🛠 Технические решения и код
### [Задача 1]
- **Проблема**: [Описание]
- **Решение**: [Какой код изменили, какие команды запускали]
- **Результат**: [Успех/ошибка]

### [Задача 2]
...

## 💬 Ключевые моменты диалога
- [Важные уточнения от пользователя]
- [Наставления или правила, принятые в ходе общения]

## 📋 Итог по файлам
- [Список всех затронутых файлов]
```

### Шаг 2: Запусти скрипт бэкапа и чистки
```bash
bash ~/freelance-2026/tools/finish_day.sh
```

### Шаг 3: Скажи пользователю
Выведи краткую сводку:
1. Что было сохранено в `chp.md`, `checkpoints/` и `reports/`
2. Что забэкаплено
3. **Инструкция**: «Для чистого старта нажми **Cmd+N** (новый чат). Затем напиши: *"Прочитай последний чекпоинт и продолжим"*»

## Важно
- Чекпоинт (`chp.md`) ДОЛЖЕН быть информативным. Новый агент в новом чате прочитает ТОЛЬКО этот файл.
- НЕ пропускай ни один шаг.
- Скрипт НЕ требует sudo и безопасен для автозапуска.
