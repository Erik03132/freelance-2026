# Архитектура Multi-Bot Engine

> Как мы строим армию ботов, не переписывая код каждый раз.

## 1. Концепция "Один Движок — Много Личностей"

Мы разделяем **Логику работы** и **Специфические знания**.

### Модуль 1: Переферия (I/O)
- **TG Adapter**: Прием/отправка сообщений в Telegram.
- **VK Adapter**: Прием/отправка сообщений в ВКонтакте.
- **Voice Module (Whisper)**: Превращает войсы в текст. 
- **TTS Module (ElevenLabs/OpenAI)**: Превращает текст бота в аудио-ответ.

### Модуль 2: Движок (The Core)
- Принимает текст запроса.
- Определяет `ProjectID` (откуда пришел запрос).
- Подтягивает нужный `System Prompt` (кто он сегодня? Ульяна или Шерлок?).
- Идет в соответствующую **Vector Database** (Knowledge Base) за контекстом.

### Модуль 3: Память и Контекст
- Хранит историю переписки по каждому пользователю, чтобы клиент мог сказать "А помнишь, я вчера спрашивал про ООО?.." и бот помнил.

## 2. Структура Баз Знаний (Knowledge Bases)

| Проект | База знаний (Контент) | Задачи |
|---|---|---|
| **AI Grant Consult** | Регламенты грантов, Налоговый кодекс РФ, Кейсы Ульяны | Квалификация, Ответы на сложные вопросы по грантам |
| **AI Bureau** | Наши тарифы, услуги, портфолио, скрипты продаж | Продажа разработки, Консультация по GEO/AIO |
| **White Label (Салоны и др.)** | Прайсы клиента, расписание мастеров, FAQ клиента | Запись на услуги, разгрузка администратора |

## 3. План реализации

1.  **Создать "Core API"**: Сервер, который принимает текст и ContextID, возвращает ответ.
2.  **Настроить интеграцию с Vector Store**: Хранилище, где лежат PDF и документы каждого проекта.
3.  **Развернуть первого пилотного бота**: Это будет наш "Грант-Эксперт" (AI Grant Consult) для Ульяны.
