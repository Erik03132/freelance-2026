# 🏛️ PROJECT CONSTITUTION: AI GRANT CONSULT

## Article I: The Core Architecture (Trinity Hub)
- Every feature must align with the **Trinity AI Editorial Hub** model:
    - **Sherl (Intel)**: Responsible for external data, GEO-scanning, and competitor analysis.
    - **Shakespeare (Editor)**: Responsible for high-quality, E-E-A-T-compliant content.
    - **Marketer (Strategist)**: Responsible for SXO/SEO visibility and technical standards.

## Article II: Technical Standards
- All code must comply with **[TECH_STANDARDS.md](../../TECH_STANDARDS.md)**.
- **Performance First**: LCP < 1.2s.
- **AI-Ready**: Every page must have proper JSON-LD and a `llms.txt` file must exist in the root.
- **SSR Over SPA**: Prefer Server-Side Rendering for maximum AI/SEO visibility.

## Article III: Spec-Driven Development (SDD)
- No feature implementation begins without a approved **Specification** in `/specs`.
- The workflow is strictly: **Specify -> Plan -> Tasks -> Implement**.
- The Specification focuses on **"WHAT"** and **"WHY"**, never "HOW".

## Article IV: Data & Security
- Primary Database: **Neon DB** (PostgreSQL).
- Real-time communication: WebSocket/Push for bot interactions.
- All secrets/keys must be in `.env.local`, never hardcoded.

## Article V: Testing & Quality
- **Test-Driven Development (TDD)**: Critical logic must be tested before implementation.
- **Integration-First**: Prefer real-world environment testing over mocks where possible.

## Article VI: Debugging & Operation
- **API Ping Protocol**: All AI keys must be verified for model compatibility before deployment.
- **Hard Reset Standard**: In case of terminal hanging or code inconsistency, the "Hard Reset" procedure (Close tab, Kill process, Clear Cache) is mandatory.
