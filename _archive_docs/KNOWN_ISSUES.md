## 🟢 РЕШЕНО: Kwork не извлекает описания задач

### Статус
✅ **Решено** — 2026-03-25

### Что было
При сканировании Kwork.ru описания задач содержали `"Фриланс маркетплейс"` (19 символов) вместо реального текста.

### Как решено
**Ключевое решение:** Вместо клика по кнопке «Показать полностью» (который падал в цикле с TimeoutError), теперь извлекаем `textContent` напрямую из **скрытого div** (`display: none`).

`textContent` в JavaScript работает даже для элементов с `display: none`, что позволяет получить полный текст без клика.

**Изменения в `src/adapters/kwork.ts`:**
- Замена поэлементного `extractText()` на batch-извлечение через `page.$$eval()`
- Приоритетное чтение из скрытого div `.wants-card__description-text > div:nth-child(2)`
- Fallback на видимый div, если скрытый пуст
- Очистка от UI-элементов ("Показать полностью", "Скрыть")

### Файлы
- `src/adapters/kwork.ts` — исправленный адаптер

## 🟢 РЕШЕНО: Шаблонные предложения без анализа

### Статус
✅ **Решено** — 2026-03-25

### Что было
Генератор предложений работал на основе ключевых слов и шаблонов (`personalized-proposal.ts`), не анализируя реальную выполнимость задачи.

### Как решено
Создан **ScoutAgent** (`src/agents/scout.ts`) — агент-аналитик, который:

1. **Определяет категорию задачи** (14 типов: web_fullstack, telegram_bot, parsing_scraping и т.д.)
2. **Оценивает выполнимость** с учётом:
   - Подключённых MCP-сервисов (Stitch, Gemini, Browser Use)
   - Навыков из `config/profile.json`
   - Блокеров (нужны credentials, юридические риски, размытое ТЗ)
3. **Рассчитывает трудоёмкость** (часы) и **анализирует вознаграждение** (₽/час)
4. **Сообщает о недостающих сервисах** и рекомендует подключение через MCP
5. **Генерирует честное предложение** — или отказ с обоснованием
6. **Самообучается** — сохраняет паттерны в `.agent/skills/`

### API Response
`POST /api/generate-response` теперь возвращает:
```json
{
  "response": "Текст предложения...",
  "metadata": {
    "source": "scout-agent",
    "analysis": {
      "category": "web_backend",
      "feasible": true,
      "effortHours": 30,
      "rewardAnalysis": { "hourlyRate": 1000, "worthIt": true },
      "skillCoverage": 75,
      "missingServices": [{ "name": "Firebase", "priority": "recommended" }],
      "blockers": []
    },
    "warnings": ["🔧 Рекомендуется подключить: Firebase MCP"],
    "operatorNotes": "📋 Категория: web_backend\n⏱ Трудоёмкость: ~30ч"
  }
}
```

### Файлы
- `src/agents/scout.ts` — Scout Agent
- `dashboard/src/app/api/generate-response/route.ts` — обновлённый API

## Другие известные проблемы

### 2. `proposals_count` не заполняется
**Статус:** ⚠️ Не реализовано

Парсеры не извлекают количество откликов. Нужно обновить селекторы:
- Kwork: `.projects__responses`
- Freelance: `.post-card__proposals`

### 3. Автоотправка работает частично
**Статус:** ⚠️ Требует `.env` с учётными данными

Функция `POST /api/auto-bid` требует:
- Скомпилированный `auto-bid.ts` в `dist/`
- Учётные данные в `.env`
- Playwright browser installed

### 4. Learning Service требует проверки
**Статус:** ⚠️ Требует тестирования

`LearningService` сохраняется в `dist/`, но может не импортироваться в dashboard из-за путей.

### 5. Визуальная генерация PDF (налоговые бланки)
**Статус:** 🛑 Anti-skill / Отказано
Попытка верстать машиночитаемые бланки ФНС через HTML/CSS или Python-оверлеи признана неэффективной. Любое смещение пикселей делает документ невалидным. Решение: переход на генерацию XML (машиночитаемые данные) для импорта в системы 1С/Контур.
