# 📜 Единый Стандарт Агента (Universal Agent Core)

Этот документ определяет минимальные технические и функциональные требования к любому ИИ-агенту в экосистеме. Каждый новый агент ОБЯЗАН наследовать эти "скиллы".

---

## 1. Стек Безопасности (Security Layer)
- **Изоляция:** Каждый агент живет в своей папке со своим `.env`.
- **Защита:** Обязательный `.gitignore` (секреты никогда не уходят в Git).
- **Ключи:** Поддержка локальных ключей и fallback на глобальные ключи воркспейса.

## 2. Стек Мозгов (Intelligence & RAG)
- **Wisdom Layer:** Статические знания (инструкции, законы, правила) хранятся в `.md` файлах. Это "базовая прошивка" агента.
- **Dynamic Layer:** Цены, товары, лог действий — в векторной базе (Neon/pgvector).
- **LLM Direct:** Использование прямых SDK (Google Gemini / OpenAI) без лишних прокси-слоев для минимальной задержки.

## 3. Стек Эффективности (Efficiency & Optimization)
- **FAQ Router (Hot Cache):** Прежде чем идти в LLM, агент проверяет `faq_cache.json`. Если вопрос типовой — ответ мгновенный и бесплатный.
- **Self-Learning (Эволюция):** Если агент дал качественный ответ через LLM, он записывает этот опыт в базу (learned knowledge), становясь умнее.

## 4. Стек Инфраструктуры (Infrastructure & Stability)
- **Connection Pooling:** Запрещено открывать новый коннект к БД на каждый запрос. Только пул (SimpleConnectionPool).
- **Health Check:** Метод `health_check()` для самодиагностики связи с БД и ИИ-моделями.
- **Memory:** Хранение истории последних 5-10 сообщений диалога для контекста.

## 5. Стек Личности (Tone of Voice)
- **Sales Layer:** Каждый ответ должен содержать экспертный "хук" или призыв к действию (soft close).
- **Эмпатия:** Тон общения задается в системных инструкциях (хозяйка, юрист, супервайзер).

---

## 🚦 Чек-лист при создании нового агента:
- [ ] Папка создана, `.gitignore` настроен.
- [ ] Подключен Neon через пул соединений.
- [ ] Написана база экспертных знаний в MD.
- [ ] Внедрена логика FAQ кэша.
- [ ] Включено самообучение (запись ответов в БД).
- [ ] Настроена память сессий.
