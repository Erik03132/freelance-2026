## MVP #1 — Базовый сбор задач (2-3 недели)

### Цель
Минимально работающий продукт: авторизация на 1-2 биржах, сбор задач, простая фильтрация, черновики писем.

### Задачи

#### Неделя 1: Настройка проекта и конфигурация

- [ ] **1.1** Создать структуру проекта
  ```
  freelance-agent/
  ├── .agent/rules/
  │   └── agent-guide.md
  ├── config/
  │   └── profile.json
  ├── src/
  │   ├── agents/
  │   ├── services/
  │   └── utils/
  ├── tasks/
  ├── prototypes/
  ├── proposals/
  └── logs/
  ```

- [ ] **1.2** Создать конфиг профиля (`config/profile.json`)
  ```json
  {
    "user": {
      "name": "",
      "nickname": "",
      "portfolio": {
        "github": "",
        "website": "",
        "dribbble": ""
      }
    },
    "skills": {
      "React": 0.9,
      "Next.js": 0.8,
      "Node.js": 0.7
    },
    "preferences": {
      "categories": ["web-development", "backend", "integrations"],
      "ignore": ["copywriting", "smm", "translation"]
    },
    "matching": {
      "skill_match_threshold": 0.7,
      "clarity_threshold": 0.6,
      "max_tasks_per_day": 5
    },
    "platforms": [
      {
        "name": "upwork",
        "url": "https://www.upwork.com/nx/search/jobs/",
        "enabled": true
      }
    ]
  }
  ```

- [ ] **1.3** Настроить систему логирования
  - Формат логов: timestamp, action, result, details
  - Ротация логов по дням

#### Неделя 2: Браузерный агент и авторизация

- [ ] **2.1** Реализовать Browser-Use модуль
  - Открытие URL
  - Навигация по страницам
  - Клик по элементам
  - Извлечение текста из DOM
  - Скроллинг с задержками

- [ ] **2.2** Реализовать модуль авторизации
  - Ввод логина/пароля (через secure input)
  - Сохранение сессионных куки
  - Проверка статуса авторизации
  - Обработка 2FA (запрос у пользователя)

- [ ] **2.3** Создать сценарии для каждой площадки
  - Upwork: логин, переход к ленте задач
  - Fiverr: логин, переход к категории услуг

#### Неделя 3: Сбор и фильтрация задач

- [ ] **3.1** Реализовать парсер задач
  - Извлечение: заголовок, описание, бюджет, сроки, теги
  - Сохранение в `tasks/raw/{platform}-{date}.json`
  - Обработка пагинации (лимит страниц)

- [ ] **3.2** Реализовать простую фильтрацию по ключевым словам
  - Точное совпадение навыков из конфига
  - Исключение стоп-слов
  - Фильтрация по категориям

- [ ] **3.3** Создать генератор черновиков предложений
  - Шаблон письма
  - Подстановка данных задачи
  - Подстановка данных профиля
  - Сохранение в `proposals/{date}-{taskId}.md`

- [ ] **3.4** Создать базовый Dashboard (CLI/Markdown)
  - Таблица задач за день
  - Статусы обработки
  - Ссылки на файлы

### Критерии готовности MVP #1
- [ ] Агент запускается по расписанию/вручную
- [ ] Успешная авторизация на 1-2 площадках
- [ ] Сбор минимум 20 задач за сессию
- [ ] Фильтрация по ключевым словам работает
- [ ] Генерация черновиков писем
- [ ] Логирование всех действий
- [ ] Отчет пользователю в конце сессии

## MVP #2 — Умный матчинг и прототипы (3-4 недели)

### Цель
Семантический матчинг навыков, оценка качества ТЗ, автогенерация прототипов.

### Задачи

#### Неделя 4: Семантический матчинг

- [ ] **4.1** Реализовать извлечение навыков из текста
  - Tokenization описания задачи
  - Выделение tech-terms (React, Node.js, API и т.д.)
  - Нормализация (синонимы: "React.js" → "React")

- [ ] **4.2** Реализовать алгоритм подсчёта skill_match_score
  - Точное совпадение (вес 0.4)
  - Семантическая близость (вес 0.3)
  - Категориальное совпадение (вес 0.3)
  - Нормализация к [0, 1]

- [ ] **4.3** Создать словарь синонимов и связанных технологий
  ```json
  {
    "React": ["React.js", "ReactJS", "React Native"],
    "Node.js": ["Node", "NodeJS", "Express"],
    "UI/UX": ["Figma", "Design System", "Prototyping"]
  }
  ```

#### Неделя 5: Оценка качества ТЗ

- [ ] **5.1** Реализовать эвристики для clarity_score
  - Наличие стека технологий (+0.2)
  - Наличие референсов/примеров (+0.2)
  - Описание пользовательских сценариев (+0.2)
  - Acceptance criteria (+0.2)
  - Детальное описание (>100 слов) (+0.2)

- [ ] **5.2** Реализовать детектор «плохих» ТЗ
  - Стоп-фразы: «как-нибудь», «красиво», «подумаем»
  - Слишком короткое описание (<50 слов)
  - Отсутствие конкретных требований

- [ ] **5.3** Интегрировать оценку в пайплайн фильтрации
  - Двухэтапная фильтрация: skills → clarity
  - Сортировка по комбинированному score

#### Неделя 6-7: Генерация прототипов

- [ ] **6.1** Создать генератор плана решения
  - Переформулировка задачи (LLM)
  - Архитектурное решение
  - Пошаговый план
  - Риски и вопросы

- [ ] **6.2** Реализовать scaffold для веб-проектов
  - Next.js шаблон (pages, components, styles)
  - Базовая верстка ключевых страниц
  - Mock-данные для демонстрации
  - Конфиг для быстрого деплоя

- [ ] **6.3** Создать шаблоны для дизайн-задач
  - Структура файлов для макетов
  - Описание концепции
  - Инструкция по генерации превью

- [ ] **6.4** Интегрировать с Dashboard
  - Статус «прототип готов»
  - Ссылка на файлы прототипа
  - Предпросмотр плана решения

### Критерии готовности MVP #2
- [ ] Семантический матчинг работает (точность >80%)
- [ ] Оценка clarity_score адекватна
- [ ] Генерация плана решения для каждой задачи
- [ ] Scaffold веб-проекта создается за <2 мин
- [ ] Dashboard показывает полные данные по задачам

## MVP #3 — Защита и масштабирование (2-3 недели)

### Цель
Меры защиты от недобросовестных заказчиков, расширение на несколько площадок, полноценный UI.

### Задачи

#### Неделя 8: Защитные механизмы

- [ ] **8.1** Реализовать водяные знаки для изображений
  - Наложение текста (logo, email, URL)
  - Настройка прозрачности и позиции
  - Пакетная обработка

- [ ] **8.2** Создать систему ограничения функциональности
  - Mock-слой для критичных функций
  - Отключение export/download
  - Ограничение по времени/доступу

- [ ] **8.3** Реализовать деплой на свой хостинг
  - Интеграция с Vercel/Netlify
  - Настройка домена
  - Автоматический деплой прототипов

- [ ] **8.4** Добавить юридические дисклеймеры
  - Шаблон текста для писем
  - README в каждом прототипе
  - Terms of Use для демо

#### Неделя 9: Расширение площадок

- [ ] **9.1** Добавить сценарии для новых площадок
  - Freelancer.com
  - Toptal
  - Специализированные биржи (Dribbble Jobs, etc.)

- [ ] **9.2** Унифицировать интерфейс парсинга
  - Общий интерфейс для всех площадок
  - Адаптеры под каждую платформу
  - Обработка разных форматов данных

- [ ] **9.3** Реализовать приоритизацию площадок
  - Настройка порядка обхода
  - Разные лимиты для разных площадок
  - Статистика по каждой площадке

#### Неделя 10: Dashboard и отчётность

- [ ] **10.1** Создать веб-интерфейс Dashboard
  - Список задач с фильтрами
  - Детали задачи (score, план, прототип)
  - Редактирование черновиков
  - Статусы и действия

- [ ] **10.2** Реализовать систему отчётов
  - Ежедневная сводка
  - Еженедельная статистика
  - Экспорт в CSV/PDF

- [ ] **10.3** Добавить режим ручной проверки
  - Пометка «интересно/не интересно»
  - Комментарии к задачам
  - Обратная связь для обучения

### Критерии готовности MVP #3
- [ ] Все защитные механизмы работают
- [ ] Поддержка 3+ площадок
- [ ] Веб-Dashboard доступен и функционален
- [ ] Еженедельные отчёты генерируются
- [ ] Режим ручной проверки работает

## Технические требования

### Минимальные
- Node.js 18+
- 2GB RAM для агента
- Доступ к браузеру (Chromium)
- 10GB свободного места

### Рекомендуемые
- Выделенный хостинг для деплоя
- PostgreSQL для хранения данных
- Redis для кеширования
- S3 для хранения прототипов

## Риски и митигация

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Блокировка на площадках | Средняя | Высокое | Человеческие задержки, лимиты запросов |
| Изменения в DOM площадок | Высокая | Среднее | Быстрое обновление парсеров, мониторинг |
| Ложные срабатывания матчинга | Средняя | Низкое | Ручная проверка, обучение на фидбеке |
| Юридические вопросы | Низкая | Высокое | Дисклеймеры, соблюдение ToS |
